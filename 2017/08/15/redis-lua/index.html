Redis结合Lua解析

2017-08-15 • 标签: redis

首先

redis作为一款优秀的缓存数据库，已成为许许多多大大小小的项目的必备底层数据库之一了，在我们使用redis时，有时会需要依赖redis来处理一段非常复杂的逻辑，而这段逻辑可能需要通过redis client发送多条redis命令来达到我们的目的，然而这种处理方式，不仅效率低，而且无法保证事务的原子性，以及可能在高并发下产生脏数据；
redis从2.6.0版本开始提供了一种新的解决方案，内置lua解释器，通过 redis Eval 命令来执行lua脚本，达到执行自定义逻辑的redis命令，并保障命令的原子性；
